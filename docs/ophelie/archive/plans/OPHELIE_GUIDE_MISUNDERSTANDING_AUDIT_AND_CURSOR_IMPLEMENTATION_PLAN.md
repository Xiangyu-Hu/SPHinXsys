### 1.1 Misunderstanding 1: Treat edge-flux as next production, not diagnostic first

Previous guide wrote “edge-conservative phi projection operator” with LHS/RHS forms. Easy to read as:

```text
Change phi equation LHS/RHS to legacy pairwise + legacy flux,
then evaluate edge-flux as new production solver.
```

More precise meaning:

```text

```

Results confirm:

```text

  divJ_L2_red ≈ 0.93    failed
```

This does not prove edge-flux fully wrong; it means: **replacing only the phi equation without E/J/divJ postprocess yields an inconsistent discrete system.**

### 1.2 Misunderstanding 2: Lower `phi_eq_res` over `divJ_L2_red`

Previous guide motivated edge-flux partly to lower `phi_eq_res`. Easy to assume:

```text
Lower phi_eq_res is always better, even if divJ_L2_red worsens.
```

Now explicit:

```text
Primary: divJ_L2_red + P_raw + fields finite + reasonable JouleHeat.

```

Current baseline:

```text

  phi_eq_res ≈ 0.486

  literature_passed = true
```

Keep baseline as production.

### 1.3 Misunderstanding 3: Vector divergence MMS pass logic not strong enough

Previous guide required vector divergence MMS but did not state:

```text
If linear_xyz L2 error≈2, check sign flip first; do not conclude “no convergence.”
```

Cursor got:

```text
linear_xyz: interior L2 ≈ 2.0, no convergence
```

Very like divergence sign opposite to exact. For:

```math
A=(x,y,z), \quad \nabla\cdot A=3
```

If discrete operator outputs `-3`, relative error ≈ 2:

```math
\frac{|(-3)-3|}{|3|}=2
```

Do not say `D_unc` fails on linear field. Add signed / flipped error diagnostic first.

### 1.4 Misunderstanding 4: `biot_like_toroidal` treated as valid Biot-like benchmark

Previous doc gave toroidal direction without stressing “must be smooth.” Current form is similar to:

```cpp
A = (-y * f / r_xy, x * f / r_xy, 0.0);
```

Field direction discontinuous on axis at `r_xy=0`; if `f(0,z) != 0`, not a smooth vector field. Real loop coil `A_theta` → 0 on axis; Cartesian components should be smooth.

Therefore non-monotonic dp convergence of `biot_like_toroidal` is not evidence “D/Dc unsuitable for Biot A.”

Use smooth toroidal field:

```cpp
A = (-y * f, x * f, 0.0);
```

With `f=f(r^2,z)`, e.g.:

```cpp
f = 1.0 + 0.1 * r_xy_sq + 0.2 * z * z;
```

Still rotational, smooth on axis, continuous divergence 0.

### 1.5 Misunderstanding 5: `compatible-div-grad` as “theoretically more correct”

`D_c/G_c` passing discrete self-consistent MMS only means:

```text
D_c and G_c are self-consistent for potential fields generated by G_c(phi).
```

Real Biot A is rotational vector potential, not from `G_c(phi)`. So `D_c(A)` is not naturally closer to continuous physical divergence.

On current French Biot A:

```text
D_unc_vs_D_c_l2 ≈ 0.97
cosine ≈ 0.38
```

They give very different numerical divergence on real Biot A. Without fixed vector divergence MMS, cannot say D_c is more correct.

## A.3 Add error metric fields

`OphelieVectorDivergenceErrorMetrics` should add:

```cpp
Real signed_l2_all = 0.0;
Real signed_l2_interior = 0.0;
Real signed_l2_boundary = 0.0;
Real flipped_l2_all = 0.0;
Real flipped_l2_interior = 0.0;

Real mean_exact_div = 0.0;
Real abs_mean_discrete_div = 0.0;
Real abs_mean_exact_div = 0.0;
```

Where:

```math
\alpha = \frac{\langle D(A), divA_{exact}\rangle}{\langle divA_{exact}, divA_{exact}\rangle}
```

Volume-weighted inner product:

```cpp
Real dot_de = 0.0;
Real dot_ee = 0.0;
for i:
    dot_de += vol[i] * discrete_div[i] * exact_div[i];
    dot_ee += vol[i] * exact_div[i] * exact_div[i];
alpha = dot_de / (dot_ee + TinyReal);
```

Also:

```cpp
signed_error = discrete_div - exact_div;
flipped_error = -discrete_div - exact_div;
best_sign_error = min(norm(signed_error), norm(flipped_error));
```

## B.4 Edge current definition

Same pair conductance as legacy pairwise LHS/RHS. Pseudo:

```math
q_{ij} = C_{ij}[(\phi_i-\phi_j) + s_A\,\omega\,\bar A_{ij}\cdot(x_i-x_j)]
```

`s_A` sign must match current `LegacyFlux` RHS convention.

Do not guess sign from memory; try:

```text
edge_res_sign_plus
edge_res_sign_minus
```

Pick lower residual consistent with current solver equation.

## B.5 Output metrics

Per particle:

```math
R_i = \sum_j q_{ij}
```

Output:

```text
edge_res_l2_level0   # phi=0
edge_res_l2_phi      # after solve
edge_res_red = edge_res_l2_level0 / edge_res_l2_phi
edge_res_linf_level0
edge_res_linf_phi
edge_res_volume_integral_level0
edge_res_volume_integral_phi
edge_pair_antisymmetry_error
```

`edge_pair_antisymmetry_error` checks:

```math
q_{ij}+q_{ji}\approx0
```

```math
\sum_i R_i V_i \approx 0
```

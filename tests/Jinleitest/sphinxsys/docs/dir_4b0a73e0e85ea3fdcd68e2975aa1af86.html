file  ownership.h [code]
 Here, the classes related to the ownership of objects are defined by implementing Resource Acquisition Is Initialization (RAII). The basic idea in SPHinXsys to use defined ownership. Unique pointers are used as much as possible and shared pointer only used when a default smart pointer is generated as argument in the constructors. In this case, for objects with statically defined lifetime, references will be used. For those with dynamical lifetime, we will use smart-pointer keepers to handel. Basically, new a raw pointer is forbidden through out the code except for raw matrixes, such as the cell linked list mesh, that will be deleted in destructor, especially, when new features are introduced. Generally, the classes here are private member and the smart pointer is used only when the memory is allocated. After that, we take reference or raw pointer out and use them as observer.
